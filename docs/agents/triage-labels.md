# Triage Labels

Apply these labels using the tracker's native label system. Never invent labels; strictly use this vocabulary.

## Triage Roles

- `needs-triage` — newly created or updated, waiting for the tech lead to route it
- `needs-info` — waiting on the author or a stakeholder to clarify requirements
- `ready-for-agent` — fully specified, scoped, and ready for an agent to pick up
- `ready-for-human` — fully specified, scoped, but requires human context or secrets to complete
- `wontfix` — rejected or obsolete; close the issue

## Additional Classifications

- `type:feature` — new functionality
- `type:bug` — bug fix
- `type:refactor` — code refactoring
- `type:security` — security fix or improvement
- `type:performance` — performance improvement

- `priority:p0` — critical, blocking
- `priority:p1` — high priority
- `priority:p2` — normal priority

- `area:web` — frontend web
- `area:api` — backend API / gateway
- `area:database` — database / schema
- `area:mobile` — mobile app (if applicable)
- `area:ai` — AI / Agent specific tasks
