# Sprint 0.1: Flow Inventory

This document maps the exact status of the system broken down by **Business Capabilities** rather than by service folder, ensuring a clear path to production readiness.

## 1. Core Identity & Accounts

### Authentication & Customer Registration
*   **Current Status:** Functional
*   **Entry Point:** `apps/identity-service/src/modules/auth/routes/auth.routes.js`
*   **Files:** `auth.controller.js`, `auth.service.js`, `user.repository.js`
*   **APIs:** `POST /api/auth/register`, `POST /api/auth/login`, `GET /api/auth/profile`
*   **Tables:** `users`
*   **Events:** None
*   **Dependencies:** `pg` (Database), `jsonwebtoken`, `bcryptjs`
*   **Technical Debt:** Raw SQL inline instead of ORM. Password hashing is directly in service. No domain events published on user creation.
*   **Feature Readiness Score:** 90%
*   **Reuse Decision:** **Reuse (95%)** - Needs a `UserCreated` event.

### Merchant Registration
*   **Current Status:** Partial
*   **Entry Point:** `apps/identity-service` (`POST /api/auth/register` with role='merchant')
*   **Files:** `auth.service.js`, `user.repository.js`
*   **APIs:** `POST /api/auth/register`
*   **Tables:** `users`
*   **Events:** None
*   **Dependencies:** Identity Service
*   **Technical Debt:** Missing integration with `restaurant-service`. Merchant is created in `identity` but no Restaurant profile is seeded.
*   **Feature Readiness Score:** 40%
*   **Reuse Decision:** **Extend (60%)** - Must trigger `MerchantRegistered` event to seed Restaurant profile.

## 2. Catalog & Restaurant

### Restaurant, Menu, & Category Management
*   **Current Status:** Mostly Read-Only
*   **Entry Point:** `apps/restaurant-service/src/modules/restaurant/routes/`
*   **Files:** `restaurant.routes.js`, `category.routes.js`, `menu-item.routes.js`
*   **APIs:** `GET /` (restaurants), `GET /:id/categories`, `GET /:id/menu`
*   **Tables:** `mer_restaurants`, `mer_categories`, `mer_menu_items` (Implied prefixes based on ERD)
*   **Events:** None identified.
*   **Technical Debt:** `restaurant-service` lacks POST routes for the Restaurant itself (assumes manual DB population). `food-service` exists and seems to duplicate this domain.
*   **Feature Readiness Score:** 50%
*   **Reuse Decision:** **Refactor (50%)** - Delete `food-service`. Implement CRUD operations in `restaurant-service`.

### Checkout
*   **Current Status:** Functional but Architecturally Flawed
*   **Entry Point:** `apps/order-service/src/modules/checkout/routes/checkout.routes.js`
*   **Files:** `checkout.controller.js`, `checkout.service.js`, `checkout.repository.js`
*   **APIs:** `POST /api/checkout/checkout`
*   **Tables:** `ord_orders`, `ord_order_items`, **`inventory_stock` (Violation)**
*   **Events:** `OrderCreated` published via Outbox.
*   **Dependencies:** `CartService`, `RestaurantGateway`, `PricingService`
*   **Technical Debt:** Critical Cross-Domain Violation. `checkout.service.js` directly executes `SELECT ... FOR UPDATE` on `inventory_stock`.
*   **Feature Readiness Score:** 75%
*   **Reuse Decision:** **Refactor (50%)** - Strip out synchronous inventory checking. Implement Saga.
