# 1. Introduction and Goals

## 1.1 Requirements Overview
FoodieGo is an online food delivery platform built on a microservices architecture. It connects customers, merchants, and administrators. 
The main goal is to provide a highly scalable, fault-tolerant, and responsive system for browsing food, managing carts, and processing orders.

## 1.2 Quality Goals
1. **Scalability**: Handle high traffic during peak meal times.
2. **Availability**: The system must remain functional even if non-critical services degrade.
3. **Consistency**: Financial transactions and orders must be strictly consistent.

## 1.3 Stakeholders
- **Customers**: Browse and order food.
- **Merchants**: Manage menus and receive orders.
- **Admins**: Platform management.
