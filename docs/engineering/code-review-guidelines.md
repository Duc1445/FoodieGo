# Code Review Guidelines

Inspired by Google's Engineering Practices.

## 1. Standard of Code Review
The primary purpose of code review is to ensure that the overall code health of FoodieGo is improving over time. Code must not be perfect, but it must be better than it was.

## 2. What to look for
- **Design**: Does the code fit the overall architecture? Is it over-engineered?
- **Functionality**: Does it do what the author intended? Is it good for the users?
- **Complexity**: Is it too complex? Could it be simplified?
- **Tests**: Does it have appropriate automated tests?
- **Naming**: Are variables/functions named clearly?
- **Comments**: Are comments explaining *why*, not *what*?

## 3. Speed of Review
Reviewers should prioritize unblocking others. If a PR is large, ask the author to split it.
