# Design Document Guidelines

Before building any significant feature or system change, write a Design Document (or ADR).

## 1. Purpose
To align stakeholders and engineers on the *Why* and *How* before writing code.

## 2. Structure
- **Context & Scope**: What are we building and why?
- **Goals & Non-Goals**: What is explicitly out of scope?
- **Design Proposal**: How will we build it?
- **Alternatives Considered**: What else did we think about?
- **Cross-cutting Concerns**: Security, performance, observability.
